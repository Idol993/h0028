## 1. 产品概述

健身房团课预约与签到核销系统，面向健身房会员、教练和管理员，提供团课浏览预约、签到核销、爽约管理、出勤统计和运营报表等全流程数字化管理，解决传统纸质签到效率低、爽约频发、排课缺乏数据支撑等痛点。

- 目标用户：健身房会员（预约上课）、教练（管理课程与签到）、管理员（运营管理与数据决策）
- 核心价值：降低爽约率、提升排课科学性、实现会员课时精细化管理

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册/登录方式 | 核心权限 |
|------|-------------|---------|
| 会员 | 手机号注册登录 | 浏览课表、预约课程、取消预约、查看个人记录与课时包、导出上课明细 |
| 教练 | 管理员创建账号 | 查看自己带的课程、扫码核销、标记到场人数、查看到场名单 |
| 管理员 | 系统预设超级账号 | 管理全部课程、教练、会员、规则设置，查看运营统计与报表 |

### 2.2 功能模块

1. **登录页**：角色选择、手机号/密码登录
2. **会员-课表页**：本周课表浏览、课程详情、一键预约/取消
3. **会员-个人中心页**：预约记录、剩余课时包、上课明细导出
4. **教练-课程管理页**：今日/本周课程列表、到场名单、扫码核销
5. **教练-课后录入页**：标记实际到场人数
6. **管理员-课程管理页**：全部课程增删改查、排课日历
7. **管理员-教练管理页**：教练账号管理、带课统计
8. **管理员-运营统计页**：出勤率排行、爽约名单、课程类型标黄预警
9. **管理员-规则设置页**：爽约阈值、预约提前时间、课时扣减规则
10. **管理员-报表导出页**：月度运营报表导出

## 3. 核心流程

### 3.1 预约与签到核销流程

会员浏览本周课表 → 选择课程点击预约 → 系统锁定席位并扣减剩余名额 → 会员可于开课前2小时取消（释放席位） → 开课前2小时未取消则自动扣一次课时 → 到场后前台/教练扫会员码核销签到 → 课程结束后教练标记实际到场人数 → 系统对比预约数与到课数，计算出勤率

```mermaid
flowchart TD
    "浏览课表" --> "选择课程预约"
    "选择课程预约" --> "锁定席位/扣名额"
    "锁定席位/扣名额" --> "是否开课前2小时?"
    "是否开课前2小时?" -->|"是,可取消"| "取消预约/释放席位"
    "是否开课前2小时?" -->|"否,不可取消"| "自动扣课时"
    "自动扣课时" --> "到场签到"
    "到场签到" --> "扫码核销"
    "扫码核销" --> "教练标记到场人数"
    "教练标记到场人数" --> "系统对比出勤率"
    "取消预约/释放席位" --> "浏览课表"
```

### 3.2 爽约处理流程

课程开始 → 会员未核销且未取消 → 系统标记为爽约 → 累计当月爽约次数 → 达到3次 → 暂停下周预约权限 → 下周结束后自动恢复

```mermaid

    "课程开始" --> "会员未核销且未取消?"
    "会员未核销且未取消?" -->|"是"| "标记爽约"
    "会员未核销且未取消?" -->|"否"| "正常完成"
    "标记爽约" --> "累计当月爽约次数"
    "累计当月爽约次数" --> "当月爽约≥3次?"
    "当月爽约≥3次?" -->|"是"| "暂停下周预约权限"
    "当月爽约≥3次?" -->|"否"| "继续正常预约"
    "暂停下周预约权限" --> "下周结束自动恢复"
```

### 3.3 出勤率预警流程

课程结束 → 教练标记到场人数 → 系统计算单课出勤率 → 汇总课程类型出勤率 → 低于50%标黄 → 提示管理员调整排课

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深碳灰(#1A1A2E) + 活力橙(#FF6B35) 作为强调色，传达力量与活力
- 辅助色：薄荷绿(#00E5A0) 表示成功/可用状态，警示黄(#FFB800) 表示预警，柔粉(#FF4757) 表示爽约/满员
- 按钮风格：圆角(8px)、略带阴影的实心按钮，hover时轻微上浮
- 字体：标题用 Outfit（几何感、运动感），正文用 Noto Sans SC（中文友好）
- 布局风格：左侧导航栏 + 右侧内容区，卡片式内容组织，课表用时间轴网格布局
- 图标风格：线性图标（Lucide），统一2px描边
